Implementation of a FL poisoning and defense system in an IoT based health prediction network

Implemented a Federated Learning (FL) system tailored for enhancing model security and accuracy in healthcare predictions. Federated Learning enables collaborative model training across distributed devices while preserving data privacy. This project focused on defending against poisoning attacks in both IID and Non-IID data distributions, crucial for maintaining model integrity in sensitive healthcare environments.

  • Features:
    • FL System Architecture: Developed a scalable FL system with a client-server model using Flask, facilitating collaborative model training while maintaining data privacy.
    • Real-time Monitoring: Implemented a web-based dashboard to monitor and visualize model accuracy and system performance, enhancing transparency and accountability.
    • FLAME Integration: Integrated the FLAME algorithm to detect and mitigate model poisoning attacks effectively, ensuring the integrity and security of federated learning models.
  • Technologies Used:
    • Python: Flask, TensorFlow, PyTorch
    • Web Development: HTML, CSS, JavaScript
    • Data Handling: Pandas, NumPy
    • Machine Learning: Federated Learning, CNNs, Gradient Descent
  • Challenges:
    • Data Heterogeneity: Managed non-IID data distributions, where each client's data differs significantly, complicating model aggregation.
    • Model Poisoning Attacks: Developed defenses to prevent malicious clients from injecting poisoned data during model updates, ensuring robustness and reliability of the FL system.
    • Privacy Compliance: Implemented privacy-preserving techniques to adhere to healthcare data regulations, safeguarding patient information during model training and aggregation.
  • Achievements: A significant achievement of this project was the successful development and implementation of a robust defense mechanism for federated learning. Our FLAME (Federated Learning with Adversarial Model Evaluation) algorithm demonstrated exceptional resilience in handling scenarios with over 50% malicious clients, maintaining model integrity and thwarting adversarial attacks effectively. Particularly noteworthy was its performance in non-IID (non-identically distributed) data environments, where it mitigated data distribution challenges and improved model accuracy by 15% compared to traditional FL methods.
